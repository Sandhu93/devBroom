# Code of Conduct

## Our Standard

This project aims to keep collaboration respectful, practical, and focused on constructive technical discussion.

Examples of behavior that contribute to a positive environment include:

- being respectful in disagreements
- giving actionable feedback
- assuming good intent
- keeping discussion focused on the code, behavior, and user impact
- welcoming contributors with different levels of experience

Examples of unacceptable behavior include:

- personal attacks or insults
- harassment or discriminatory language
- bad-faith arguments or deliberately disruptive behavior
- dismissive or hostile responses to contributors asking questions
- publishing private information without permission

## Scope

This Code of Conduct applies to:

- issues
- pull requests
- discussions
- project communication spaces related to this repository

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, or pull requests that do not align with this Code of Conduct.

Repeated or severe violations may result in temporary or permanent exclusion from the project’s collaboration spaces.

## Reporting

If you run into unacceptable behavior, contact the maintainer through GitHub or the contact method listed on the repository profile.

## Notes

This is a lightweight conduct policy intended for a small open repository. The goal is to keep collaboration professional and useful without adding unnecessary process.
